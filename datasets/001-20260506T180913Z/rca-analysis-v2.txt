## **2. Root Cause**
The **frontend service is failing to route requests properly**, leading to:
- **404 errors** (missing resources, likely due to misconfigured session storage).
- **500 errors** (backend service failures, possibly due to unavailable dependencies).
- **MemoryStore warning** (session storage misconfiguration, causing session loss).

This cascades into:
- **Catalogue service latency spike** (likely due to retries from frontend).
- **Orders & Shipping services being unreachable** (no HTTP traffic, no health checks).
- **Tempo CPU spike** (increased tracing load due to failed requests).

**Primary Root Cause:**
The **frontend service is misconfigured** (session storage in `MemoryStore` is not persistent), causing:
1. **Session loss** → **404s** (users lose session state).
2. **Failed backend calls** → **500s** (orders/shipping services are unreachable).
3. **Retry storms** → **catalogue latency spike**.

## **4. Impact**
- **End-users experience:**
  - **404 errors** (session loss, broken UI flows).
  - **500 errors** (failed checkouts, order submissions).
  - **Slow catalogue browsing** (high latency).
- **Business impact:**
  - **Lost revenue** (failed orders).
  - **Poor user experience** (session drops, errors).

---

## **5. Suggested Fix**

### **Immediate Actions**
1. **Fix frontend session storage:**
   - Replace `MemoryStore` with a **persistent session store** (e.g., Redis, database-backed).
   - Validate session persistence under load.
2. **Check backend service availability:**
   - Verify **orders** and **shipping** services are running (`up` metric is `1.0`, but no traffic).
   - Check **network policies** (are frontend → backend calls blocked?).
3. **Temporarily reduce tracing load:**
   - Lower Tempo sampling rate if CPU spikes persist.

### **Long-Term Fixes**
1. **Add circuit breakers** in frontend to prevent retry storms.
2. **Implement proper health checks** for orders/shipping (currently `java_http_2xx_health=0`).
3. **Monitor session storage** (alert on `MemoryStore` warnings).
4. **Load test** the system to validate fixes.

---

### **Summary**
**Root Cause:** Frontend session misconfiguration (`MemoryStore`) → session loss → 404s → failed backend calls → 500s → retry storms → catalogue latency.
**Fix:** Replace `MemoryStore`, validate backend availability, add circuit breakers.
